Overview of how the Methods work


How are the methods sent

The methods are sent using the common GET syntax in the url. For more information on how to use the GET method of sending data please refer to this W3C schools guide. There are many many other resources which can be found with a Google search if the W3C guide is not your favorite.

Example of the method "get_recent_datasets"

http://data.nasa.gov/api/get_recent_datasets?count=12&dev=1

Breaking this url down into its component parts we have the following separate pieces,

  • http://data.nasa.gov/api
    • This simply directs the request to the portal to the api.

  • get_recent_datasets
    • Tthis is the method you are using for this particular call. This is one of the 9 methods NASA has provided.

  • ?count=12&dev=1
    • After the ? we have two parameters we have passed to the method. The first "count" is simply to tell the method how many JSON responses to return. The second parameter "dev" is a function which returns the JSON object in a more human readable form. This is very useful for previewing the data which is being returned from the method call. There are two examples below of the JSON object with the "dev" call and without it.

Example of the data returned from the method

This is the example with the "dev" variable set to make the JSON result easier to read.

http://data.nasa.gov/api/get_recent_datasets/?count=2&dev=1

Verus the return data without the "dev" variable set to 1. As you can see this information is much more difficult to make out.

http://data.nasa.gov/api/get_recent_datasets/?count=2&dev=0

Essentially the difference is the same as a Javascript library which has been minimized. All of the spaces have been taken out of the "dev=0" version of the JSON.
